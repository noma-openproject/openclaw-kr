# 메모리 프로토콜 (섹션 3-4)

## 핸드오프 루틴 (세션 종료 시)

사용자가 "끝", "마무리", "오늘 여기까지" 등 세션 종료 신호를 보내면:
1. 현재 진행 상황 요약
2. 남은 TODO 목록
3. 오늘 내린 핵심 결정
→ CLAUDE.md에 반영 (200줄 이하 유지)

## 컨텍스트 과부하 감지 → 자동 제안

아래 신호 중 하나라도 감지되면 /compact 또는 /clear를 사용자에게 제안:
- 이미 제공된 정보를 다시 물어볼 때
- 이전 아키텍처 결정과 모순되는 코드를 생성할 때
- 파일 구조를 잘못 기억할 때
- 코드 스타일이 점점 일관성을 잃을 때

## /compact vs /clear 판단

| 상황 | 조치 |
|---|---|
| 이전 컨텍스트가 대부분 유용 | /compact |
| 이전 컨텍스트가 전혀 불필요 | /clear |
| 실패한 접근법이 컨텍스트를 오염 | /clear |
| 도메인 전환 (백엔드→프론트엔드) | /clear |
| 유용+노이즈 혼재 | /compact "keep X, discard Y" |
